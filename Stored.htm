Stored

During a Free Game or Free Spin Feature, the situation might arise when we Spin up a new Free Game or Free Spin combination. The decision to commence the new Feature or continue the old one is made via the Free Games or Free Spins (2) Tabs in the Picture Properties Screen.

If we wish to continue play of the old Feature then a decision has to be made as to whether the new Feature should be Stored for playback later. If there is more than one such emergent Feature, how many can be Stored? The maximum, or cap for each Picture under consideration is 10 per Payline and the default setting is 5. Newly Stored Features always appear last in the "queue" and any surplus to the cap are ignored.

In a Multiline game, the middle Payline is always considered first, the bottom Payline, when active, is always considered last. No further combinatons in the same Spin are Stored after a Feature-Reset has been detected. Scatter Features are treated as middle Payline combinations.

When a feature is Stored, the exact positions of the Reels that hold the Pictures with the Feature are queued, to be called upon at cessation of the previous plays. Queue position is, of course, determined by the settings explained in Free Game & Free Spin Contentions.

Messages in the Game Messages Area on the Game Screen provide relevant information pertaining to the above but there is no visual status on current queue sizes in this Version.
