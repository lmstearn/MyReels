OpenError

  • The issue is reproduced by first dragging Pictures into some of the RHS boxes in the Configure New Pictures Screen.
  • Click "Allocate" to assign those Pictures to the current directory, let's call it A.
  • Then click "Open & Create a new Save Directory," and save the new Profile Configuration to Directory B.
  • An error is generated when Directory A is later accessed either by double-clicking Slotdata.s$t in an explorer window, a directory change from the Game Screen, or clicking the profile name through the Start Menu.

The problem is because "Allocate" does not update the current Picture count to Slotdata.s$t in Directory A, so a smaller selection count is not compatible.

A user resolution to the issue can be any combination of the following:

  • Having chosen the required number of Pictures, click "Open & Create a new Save Directory," for Directory B, and then click the "Allocate" button.
  • Use as many Pictures as there were in the current (Directory A) Profile Configuration.
  • Before running the game, use Windows Explorer to manually create or copy, then rename, the missing "nn.bmp"s in Directory A.
  • Click "Allocate" or "Next" to overwrite the current (Directory A) Profile Configuration without creating a new save Directory B. This, of course, will overwrite the current Profile Configuration if that was the intention.
Notes:
There is never any issue with Directory B.
There is a tooltip on the "Open & Create a new Save Directory" button as a reminder.
This is a little awkward to address programatically. The original intention of the "Allocate" button was to anneal the antedeluvian navigation provided by the LH directory tree.
