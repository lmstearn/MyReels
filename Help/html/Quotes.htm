Quotes

There is a facility on the Text Tab of the General Options Screen to create a Quotes Database with the option of Quote Thumbnails. The requirement is an ASCII encoded file containing text separated by carriage returns.

Any lines of text longer than 172 characters (on a Win 95 machine, longer text lines created clipping of certain fonts in the native 800 X 600 resolution), will be automatically truncated. No more than 32767 quotes allowed, according to scroll limitations in the Configure Thumbnails Screen. The Database stores its records in alphabetic sequence based on the initial character of each quote.

Filesizes and available space appear as a Tooltip on the "Select Quotes file" button.

If there are issues with either the text file or Database a notification duly pops up indicating cause of the Quotes generation failure.

When a new Profile Configuration is created in a different directory to the install directory, an empty Quotes.s$t is also created, Filesizes and available disk space appear as a Tooltip on the "Open/Create a new Save Directory" and "Allocate Pictures to ...." buttons on the Configure New Pictures Screen.

It is not recommended to have a custom Quote Database in the Base Directory, as it is overwritten when MyReels is updated, re-installed or repaired.
Notes for Quotes bundled at Installation:
The Quotes file in the MyReels base directory is a shared resource.
Direct modifications to the Database are done through the Configure Thumbnails Screen from any Profile Configuration that uses it.
A new or different Profile Configuration can also access the file by checking the Use Base Dir. option.
This Profile Configuration also keeps a tally of the total number of records in the Quotes Database.
To update this tally, any incremental changes to the Quotes file undertaken by external processes e.g. another Profile Configuration or Updates, require the disabling, and then re-enabling of the Use Base Dir. check.
The Quotes file uses a diverse range of Sources on the WWW.
