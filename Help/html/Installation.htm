Installation

Installation
The installer for Version 3.X is recommended. To check for updates to the program use the Updater.
If intending to upgrade to 3.X, uninstalling older versions of MyReels will not remove custom Profile Configurations created post installation as those can still be used for the new Version.
Running an installer newer than 3.0 with MyReels already installed invokes a repair dialog. The behaviour of Repair is as if the game was installed for the first time.
The newer version of Active Candy buttons in MyReels will malfunction if OLE automation is absent, obsolete or incorrectly installed. Please contact The Author if this scenario is suspect.
The installation for Version 3.X is per-user, but the MyReels System Files install is not, so it is considered as a separate, albeit child install. Multi-user systems can resource a shared game installation directory, but it is not recommended without Uninstallation considerations.
Uninstallation
Locate the MyReels entry in Apps & Features or Programs & Features (used to be double-clicking the Add/Remove Programs icon in Control Panel and click Add/Remove). This is preferred to executing unins000.exe or unins001.exe from the base directory as certain uninstall scripts do not run.
For complete removal of Versions older than 3.X, any other Profile Configurations that you have created that exist entirely in directories you have nominated, will have to be deleted manually; and their associated Shortcuts will need to be deleted from the Programs group in the Start Menu. In Version 3X greater flexibility is introduced when prompted for the deletion of extra files in specific locations.
If MyReels Versions prior to 2.20 are not uninstalled prior to install of 2.2X as per recommendation, users will be left with a redundant, (as far as MyReels is concerned) registered ACTCNDY4.ocx in any &winsys& ( Windows System or System32 directory). If such a file exists on your machine (even If such an old version of MyReels were installed on any system later than Win2k) please refer to ActiveX Maintenance.
It is recommended to uninstall MyReels system files when prompted during the uninstall.
When uninstalling MyReels from a shared directory on a multi-user system, it is not recommended to delete any extra directories not created by you, when prompted, without prior consent from the other users.
As MyReels 2.2X and later uses VB6 runtimes, uninstalling prior Versions is recommended (particularly on older machines: see Note below) in removing older installed components like VB5 runtimes & MSflexgrd.ocx.

If there are issues with getting the game to work consider the following in a bug report:

  • Windows Version
  • User Access: NT Logon rights or UAC issues.
  • Is the install directory pathname 100 characters or less?
  • Was there any unusual system activity during or after the install?
  • Did the install procedure work correctly and the Finish button clicked?
  • Were other programs running while starting MyReels?
  • Was the game launched from the Start menu?
  • Did the Splash or Intro Screen or the Game Screen ever show?
  • The kind of error generated and any information provided with error message numbers.
  • Old Installer: Possibility of a corrupted(?) later version of one of the packaged System install files. To check, use a Text Editor to view the install.log file located in the directory where the game is installed, the MyReels base directory, verifying if any files were not copied to your &winsys& directory due to a (previous) "version" condition.
  • Last Resort: Uninstall MyReels, restart Windows and install again by running MyReels.exe and following the install instructions etc..
